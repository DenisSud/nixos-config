# NixOS Configuration

This repository contains my NixOS system configuration, supporting multiple machines (PC and ROG Zephyrus G14 laptop).

Dotfiles are managed as separate git repos and symlinked into place:

- **[dotfiles-shell](https://github.com/DenisSud/dotfiles-shell)** — Fish + Starship + Ghostty
- **[dotfiles-nvim](https://github.com/DenisSud/dotfiles-nvim)** — Neovim config
- **[dotfiles-pi](https://github.com/DenisSud/dotfiles-pi)** — Pi coding agent config

## Repository Structure

```
nixos-config/
├── flake.nix                      # Flake entry point, defines nixosConfigurations
├── flake.lock                     # Locked dependencies
├── configuration.nix              # Base system configuration (shared by all hosts)
├── AGENTS.md                      # This file
└── modules/
    ├── pc-config.nix              # Desktop PC specific config
    ├── pc-hardware-configuration.nix
    ├── g14-config.nix             # Laptop (ASUS ROG G14) specific config
    └── g14-hardware-configuration.nix
```

## Machines

### `pc` (Desktop)
- Hostname: `pc`
- GPU: NVIDIA RTX 5070 (Blackwell) with open kernel module
- Uses systemd-boot
- Ollama exposed to LAN

### `g14` (Laptop)
- Hostname: `g14`
- GPU: AMD Cezanne (integrated) + NVIDIA RTX 3050 Mobile (hybrid graphics)
- Uses NVIDIA Prime offload mode
- ASUS-specific hardware (potentially supergfxd)

## Applying Configuration

### Build (dry-run)
```bash
sudo nixos-rebuild dry-build --flake .#pc
```

### Switch
```bash
sudo nixos-rebuild switch --flake .#pc
# or for laptop
sudo nixos-rebuild switch --flake .#g14
```

### Using `nh` (Nix Helper)
```bash
# From any directory, nh reads the flake path from config
nh os switch
nh os boot
```

### Update flake inputs
```bash
sudo nix flake update
```

## Dotfiles Setup

After a fresh install, clone and link the dotfile repos:

```bash
# Clone repos
git clone git@github.com:DenisSud/dotfiles-shell.git ~/.dotfiles-shell
git clone git@github.com:DenisSud/dotfiles-nvim.git ~/.dotfiles-nvim
git clone git@github.com:DenisSud/dotfiles-pi.git ~/.dotfiles-pi

# Symlink everything
fish ~/.dotfiles-shell/fish/dotfiles.fish link
```

The `dotfiles.fish` script creates these symlinks:

| Source | Target |
|--------|--------|
| `~/.dotfiles-pi/settings.json` | `~/.pi/agent/settings.json` |
| `~/.dotfiles-pi/models.json` | `~/.pi/agent/models.json` |
| `~/.dotfiles-pi/themes` | `~/.pi/agent/themes` |
| `~/.dotfiles-pi/skills` | `~/.pi/agent/skills` |
| `~/.dotfiles-pi/extensions` | `~/.pi/agent/extensions` |
| `~/.dotfiles-nvim` | `~/.config/nvim` |
| `~/.dotfiles-shell/fish/config.fish` | `~/.config/fish/config.fish` |
| `~/.dotfiles-shell/starship.toml` | `~/.config/starship.toml` |
| `~/.dotfiles-shell/ghostty/config` | `~/.config/ghostty/config` |

## Secrets

User-specific secrets (API keys, tokens) are stored in `~/.env` and sourced by Fish shell. This file is NOT tracked in git.

```fish
# ~/.env example:
set -gx OPENAI_API_KEY "sk-..."
```

## Common Tasks

### Add a new system package
Edit `configuration.nix` → `environment.systemPackages` list.

### Add a new user package
Edit `configuration.nix` → `users.users.denis.packages` list.

### Add a new host
1. Generate hardware config: `sudo nixos-generate-config --flake .#hostname`
2. Create `modules/hostname-config.nix` for host-specific settings
3. Add to `flake.nix` → `nixosConfigurations`

### Rebuild after hardware changes
```bash
sudo nixos-generate-config --flake .#pc --show-hardware-config > modules/pc-hardware-configuration.nix
```

## Notes

- Both machines share `configuration.nix` as base
- Host-specific configs are in `modules/*-config.nix`
- Pi agent uses declarative config only; runtime state (sessions, auth) is per-device
- `nh` runs weekly garbage collection automatically
- Dotfiles are managed as independent git repos, not through NixOS
